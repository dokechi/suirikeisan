# 水理計算シンプルMVP

給水装置・簡易水理計算のための**静的Webアプリ**です。GitHub Pages でそのまま公開できる構成にしてあります。

今回のMVPでは、ユーザー提供のPDF資料をもとに、まず実務で使いやすい範囲だけを切り出しました。

- 同時使用水量の概算
  - 戸数式
  - 居住人数式
  - 標準化法（13/20/25mm器具数、または全使用水量＋器具数の直接入力）
- 摩擦損失水頭の計算
  - Weston 公式（口径 50mm 以下）
  - Hazen-Williams 公式（口径 75mm 以上）
- 所要水頭と圧力の適否判定
  - 分岐点最小動水圧 0.15 MPa
  - 最大静水圧 0.74 MPa
- 候補口径の総当たりによる概算選定

## このリポジトリの狙い

いきなり管網解析やポンプ選定までやるのではなく、まずは次の流れを1本にしたMVPです。

1. 同時使用水量を出す
2. 想定口径で摩擦損失を出す
3. 配水管側の圧力・高低差・器具条件を入れて適否判定する
4. 候補口径を一括比較して最小適合口径を提案する

## できること

### 1) 同時使用水量

- **戸数式**
  - 10戸未満: `Q = 42 * N^0.33`
  - 10戸以上 600戸未満: `Q = 19 * N^0.67`
- **居住人数式**
  - 30人以下: `Q = 26 * P^0.36`
  - 31人以上 200人以下: `Q = 13 * P^0.56`
- **標準化法**
  - `Q = 全使用水量 / 総給水用具数 * 使用水量比`
  - 13/20/25mm の標準使用水量を内蔵
  - 使用水量比は表値を内蔵し、11〜14・16〜19・21〜29 は線形補間

### 2) 摩擦損失

- **Weston 公式**: 小口径（50mm 以下）
- **Hazen-Williams 公式**: 75mm 以上
- 流速、損失水頭、100m当たり損失水頭を表示
- 参考として **流速 2.0 m/s 以下**の注意表示

### 3) 所要水頭・圧力判定

- 分岐点の動水圧を水頭換算して利用可能水頭を算出
- 高低差、器具必要水頭、水道メータ損失、局部損失、摩擦損失を合算
- 余裕水頭を表示
- 静水圧を入力した場合は 0.74 MPa 基準もチェック

### 4) 口径概算

- 候補口径をカンマ区切りで指定可能
- 各候補について
  - 採用式
  - 流速
  - 摩擦損失
  - 総所要水頭
  - 余裕水頭
  - 判定
  を表で比較
- 最小適合径を自動で提案

## ローカル実行

依存関係はありません。最も簡単なのは次です。

```bash
python -m http.server 8000
```

その後、ブラウザで `http://localhost:8000` を開いてください。

## テスト

Node.js がある場合は、計算ロジックだけ簡単に確認できます。

```bash
npm test
```

## GitHub Pages 公開

手順は `docs/publish-to-github.md` にまとめています。最短だと次です。

```bash
git init
git add .
git commit -m "Initial commit"
git branch -M main
git remote add origin https://github.com/<YOUR_NAME>/hydraulic-simple-calc.git
git push -u origin main
```

その後、GitHub の Settings -> Pages で `Deploy from a branch` / `main` / `/root` を選べば公開できます。

## 使うときの注意

- これは**概算・一次判定用**です。最終設計・申請・承認用ではありません。
- 水道メータ損失、器具必要水頭、局部損失、事業体ごとの細則は必ず別途確認してください。
- PDF本文や図表そのものは、このリポジトリに含めていません。権利確認なしにそのまま再配布しないでください。
